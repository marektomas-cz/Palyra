Operate the browser service and browser-backed automation sessions

Usage: palyra browser [OPTIONS] <COMMAND>

Commands:
  status       Show browser service health, policy, and local lifecycle state
  start        Start a local browser service in the background
  stop         Stop the local browser service started by this CLI
  open         Create a session and immediately navigate it to a URL
  session      Manage browser sessions
  profiles     Manage browser profiles
  tabs         Manage browser tabs for a session
  navigate     Navigate an existing browser session
  click        Click an element in the active tab
  type         Type text into an element in the active tab
  fill         Clear an element and then type text into it
  scroll       Scroll the active tab
  wait         Wait for selector or text to appear
  snapshot     Observe the active tab and emit a bounded page snapshot
  screenshot   Capture a screenshot from the active tab
  title        Read the active page title
  network      Read the bounded network log for the active tab
  storage      Inspect cookies and storage for a browser session
  errors       Inspect failed browser actions for a session
  trace        Export a trace-like debug artifact for a browser session
  downloads    List download artifacts for a browser session
  permissions  Get or set browser permissions for a session
  reset-state  Reset browser state for a session
  console      Show browser console diagnostics for a session
  pdf          Export the active browser session as PDF
  press        Press a key or key chord in the active browser session
  select       Set the selected value of a <select> element
  highlight    Temporarily highlight a selector in the active browser session

Options:
  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>
          Use the named CLI profile from the local profile registry

      --expect-profile <EXPECT_PROFILE>
          Require the resolved active profile name to match this value

      --config <CONFIG_PATH>
          Use this palyra.toml config file

      --state-root <STATE_ROOT>
          Use this runtime state root directory

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          Set diagnostic log verbosity

          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          Select text, JSON, or NDJSON output for automation

          [default: text]
          [possible values: text, json, ndjson]

      --plain
          Disable styled terminal output

      --no-color
          Disable ANSI color output

      --allow-profile-mismatch
          Allow commands to continue when the selected profile differs from context

      --allow-strict-profile-actions
          Permit strict profile-scoped actions after explicit profile checks

Examples:
  palyra browser status
  palyra browser start --wait-ms 15000
  palyra browser profiles list
  palyra browser session create --allow-domain docs.palyra.dev
  palyra browser navigate <session-id> --url https://example.com/
  palyra browser snapshot <session-id> --include-visible-text --output ./snapshot.json
  palyra browser screenshot <session-id> --output ./page.png
  palyra browser trace <session-id> --output ./trace.json

Discoverability:
  Session list/show/inspect talks directly to browserd. Mutating actions go through the control plane so policy and audit hooks stay intact.
