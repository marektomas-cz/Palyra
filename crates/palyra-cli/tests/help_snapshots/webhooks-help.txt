Manage webhook-backed integrations

Usage: palyra webhooks [OPTIONS] <COMMAND>

Commands:
  list
  show
  add
  enable
  disable
  remove
  test     [aliases: verify]

Options:
  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>
          Use the named CLI profile from the local profile registry

      --expect-profile <EXPECT_PROFILE>
          Require the resolved active profile name to match this value

      --config <CONFIG_PATH>
          Use this palyra.toml config file

      --state-root <STATE_ROOT>
          Use this runtime state root directory

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          Set diagnostic log verbosity

          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          Select text, JSON, or NDJSON output for automation

          [default: text]
          [possible values: text, json, ndjson]

      --plain
          Disable styled terminal output

      --no-color
          Disable ANSI color output

      --allow-profile-mismatch
          Allow commands to continue when the selected profile differs from context

      --allow-strict-profile-actions
          Permit strict profile-scoped actions after explicit profile checks

Examples:
  palyra webhooks list
  palyra webhooks add github_repo_a github --secret-ref global/github_repo_a --allow-event push --allow-source github.repo_a
  palyra webhooks test github_repo_a --payload-stdin

Discoverability:
  `webhooks` manages secret-aware webhook integrations without exposing a public ingress surface by default.
