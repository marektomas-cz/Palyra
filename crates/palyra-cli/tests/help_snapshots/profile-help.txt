Manage CLI connection profiles and active environment selection

Usage: palyra profile [OPTIONS] <COMMAND>

Commands:
  list
  show
  create
  clone
  export
  import
  use
  rename
  delete

Options:
  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>
          Use the named CLI profile from the local profile registry

      --expect-profile <EXPECT_PROFILE>
          Require the resolved active profile name to match this value

      --config <CONFIG_PATH>
          Use this palyra.toml config file

      --state-root <STATE_ROOT>
          Use this runtime state root directory

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          Set diagnostic log verbosity

          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          Select text, JSON, or NDJSON output for automation

          [default: text]
          [possible values: text, json, ndjson]

      --plain
          Disable styled terminal output

      --no-color
          Disable ANSI color output

      --allow-profile-mismatch
          Allow commands to continue when the selected profile differs from context

      --allow-strict-profile-actions
          Permit strict profile-scoped actions after explicit profile checks

Examples:
  palyra profile list
  palyra profile create staging --mode remote --set-default
  palyra profile clone prod staging --json
  palyra profile export prod --output ./artifacts/prod-profile.json
  palyra profile export prod --mode encrypted --password-stdin --output ./artifacts/prod-profile.enc
  palyra profile import --input ./artifacts/staging-profile.enc --password-stdin --json
  palyra profile show
  palyra profile use prod
  palyra profile rename stage staging
  palyra profile delete old-sandbox --yes

Discoverability:
  `profile show` without a name resolves the active profile.
  `profile create`, `profile clone`, and `profile import` assign isolated per-profile state roots.
  `profile clone` and `profile import` also write copied config snapshots into isolated per-profile config paths when source config is available.
