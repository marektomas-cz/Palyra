Run the ACP stdio bridge and legacy compatibility shim

Usage: palyra acp [OPTIONS]
       palyra acp <COMMAND>

Commands:
  shim  Run the legacy NDJSON ACP compatibility shim

Options:
      --grpc-url <GRPC_URL>

      --token <TOKEN>

      --principal <PRINCIPAL>

      --device-id <DEVICE_ID>

      --channel <CHANNEL>

      --session-key <SESSION_KEY>

      --session-label <SESSION_LABEL>

      --require-existing

      --reset-session

      --allow-sensitive-tools

  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>

      --expect-profile <EXPECT_PROFILE>

      --config <CONFIG_PATH>

      --state-root <STATE_ROOT>

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          [default: text]
          [possible values: text, json, ndjson]

      --plain

      --no-color

      --allow-profile-mismatch

      --allow-strict-profile-actions

Examples:
  palyra acp
  palyra acp --session-key ops:triage --session-label "Ops Triage"
  palyra acp --require-existing
  palyra acp shim --session-id 01ARZ3NDEKTSV4RRFFQ69G5FAW --prompt "hello"
  palyra acp shim --ndjson-stdin

Discoverability:
  `acp` is the preferred ACP bridge entry point. `palyra agent acp` and `palyra agent acp-shim` remain compatible.
  CLI defaults for `session_key`, `session_label`, `require_existing`, and `reset_session` seed bridge behavior; `_meta` prompt overrides still win when present.
